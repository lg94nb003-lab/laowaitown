# AI需求提问模板 - 直接复制使用

> **使用说明：** 复制下面的模板，填入你的需求，直接发给AI即可。
> 所有[方括号]的内容都需要替换成你的实际需求。

---

## 📋 完整模板（推荐）

```markdown
# 【需求标题】[用一句话描述你要做什么]

## 【现状描述】
[描述现在的情况，越具体越好]

包括：
- 功能在哪里（页面哪个位置）
- 现在是怎么工作的
- 如果有相关代码位置，也说明（如"数据配置区"）

## 【希望效果】
[描述你想要的最终效果，用日常语言，不要用技术术语]

分点说明：
1. 第一点效果
2. 第二点效果
3. 第三点效果

示例（不要照抄，仅作参考）：
1. 点击"欧洲"只显示欧洲国家
2. 加一个搜索框，可以输入"德国"快速找到
3. 选了法国显示绿色政策框，选了美国显示红色提示框

## 【特别重要的要求】
⚠️ **改完后必须检查：**
- [ ] [功能A]还能正常用（例如：签证计算器）
- [ ] [功能B]还能正常用（例如：汇率换算）
- [ ] [功能C]还能正常用（例如：中英文切换）
- [ ] 按F12看控制台，没有红色错误
- [ ] 刷新页面，所有功能都正常显示
- [ ] [其他你关心的功能]

## 【你怎么交付给我】
1. 用白话告诉我改了哪里（不要用技术术语）
2. 教我一步步测试（例如：先刷新页面，然后点XX，看XX）
3. 告诉我如果出问题怎么恢复
4. 如果涉及多个方案，请先给我选择

## 【我不懂的术语请用白话解释】
[如果有技术概念，让AI用白话解释]

要求：
- 不要说"添加onchange事件"
- 要说"让下拉框选择后自动更新结果"
- 不要说"修改DOM结构"
- 要说"改变页面的显示方式"
```

---


## 🎯 常用场景模板

### 场景1：添加/修改数据

```markdown
# 【需求标题】[添加/修改XX数据]

## 【现状描述】
现在[数据类型]有[数量]个，在代码的"数据配置区"。
位置：搜索"[数据名称]"可以找到。

## 【希望效果】
添加/修改以下内容：
1. [具体内容1]
2. [具体内容2]
3. [具体内容3]

要求：
- 按照现有格式添加
- 添加后能正常显示
- 不影响其他功能

## 【特别重要的要求】
⚠️ **改完后必须检查：**
- [ ] 相关功能正常
- [ ] 搜索功能正常
- [ ] 中英文切换正常
- [ ] 按F12没有红色错误

## 【你怎么交付给我】
1. 告诉我在哪一行添加了什么
2. 教我测试：刷新页面 → 查看新数据
3. 提供添加格式示例，方便我以后自己加
```

---

### 场景2：修改功能效果

```markdown
## 【希望效果】
改成：[描述你想要的样子]。

具体要求：
1. [要求1]

3. [要求3]

示例（不要照抄）：
- 现在：绿色背景的政策框
- 改成：蓝色背景的政策框
- 其他：文字、大小、位置都不变

## 【特别重要的要求】
⚠️ **改完后必须检查：**
- [ ] 修改的功能正常
- [ ] 不能影响[功能A]
- [ ] 不能影响[功能B]
- [ ] 按F12没有红色错误

## 【你怎么交付给我】
1. 用白话告诉我改了哪里
2. 教我测试：[具体测试步骤]
3. 告诉我如果效果不满意怎么调整
```

## 💡 填写技巧

### 技巧1：现状描述要具体

❌ **不好：**
"免签功能有点问题"

✅ **好：**
"免签入境功能，点击'全部'标签后，国家列表不显示。按F12看到红色错误：Cannot access before initialization。这个问题是上次修改国家选择方式后出现的。"

---

### 技巧2：希望效果用白话

❌ **不好：**
"给select标签添加onchange事件"

✅ **好：**
"让用户切换货币时，自动更新兑换结果，不用点'获取汇率'按钮"

---

### 技巧3：不能影响要列全

❌ **不好：**
"不能影响其他功能"

✅ **好：**
"不能影响：
- 签证计算器
- 时区转换器
- 汇率换算器（手动点击按钮时）
- 中英文切换
- 240h过境免签功能"

---

### 技巧4：测试步骤要详细

❌ **不好：**
"测试一下功能"

✅ **好：**
"教我测试：
1. 刷新页面 → 应该看到XX
2. 点击XX → 应该看到XX
3. 输入XX → 应该看到XX
4. 切换英文 → 应该看到XX
5. 按F12 → 应该没有红色错误"

## 🎯 核心要点

### 记住这个公式

```
好的需求描述 = 清晰的现状 + 明确的效果 + 完整的约束 + 详细的测试
```

### 记住这个流程

```
1. 复制模板
2. 填入需求
3. 发给AI
4. 等待修改
5. 按步骤测试
6. 验收通过/要求重做
```

### 记住这5句话

1. **说效果，不说代码**
2. **说清楚不能影响什么**
3. **要求白话解释**
4. **要求测试步骤**
5. **要求撤销方法**
