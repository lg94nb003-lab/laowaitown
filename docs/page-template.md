# LAOWAITOWN Page Template

Use this structure for new SEO and policy pages. The goal is simple: a traveler should get a clear answer first, then see conditions, risks, tools, and official references.

## Required page structure

1. One-sentence answer.
2. Who this page is for.
3. Conditions checklist.
4. Step-by-step action list.
5. Common mistakes or refusal risks.
6. FAQ.
7. Related LAOWAITOWN tools.
8. Official sources or reference notes.
9. Next recommended page.

## Required technical fields

Each public page should include:

- `<title>` with the specific query target.
- Meta description.
- Canonical URL.
- Article JSON-LD.
- FAQ JSON-LD when the page has FAQ content.
- Internal links back to core tools.
- Source/trust module.

## Recommended content style

- Start with the answer, not background.
- Use plain English for foreign travelers.
- Avoid legal certainty when policy depends on port, airline, or officer handling.
- Use "may be eligible" when conditions must be checked.
- Use "verify with official sources before travel" on policy-sensitive pages.

## Directory pattern

- Core visa directory: `/visa/`
- Country pages: `/visa/china-visa-free-entry-for-{country}-citizens.html`
- Port pages: `/visa/{city}-240-hour-transit.html`
- Route pages: `/visa/{origin}-{city}-{destination}-240-hour-transit.html`
- Life pages: `/life/{topic}.html`

## Current generation workflow

The first SEO matrix is generated by:

```bash
npm run generate:seo
```

After generation, run:

```bash
npm run check
```
