## 使用问题

### Q1：如何打开项目？

**问题：** 我不知道怎么打开 index.html 文件

**解决方案：**

```
方法1：双击打开
- 找到 index.html 文件
- 双击文件
- 系统会自动用默认浏览器打开

方法2：拖拽到浏览器
- 打开浏览器
- 把 index.html 文件拖到浏览器窗口

方法3：右键打开
- 右键点击 index.html
- 选择"打开方式"
- 选择浏览器（Chrome、Edge等）
```

---

### Q2：页面显示不正常怎么办？

**问题：** 打开后页面样式混乱或功能不正常

**解决方案：**

```
1. 强制刷新页面
   - Windows：Ctrl + F5
   - Mac：Cmd + Shift + R

2. 清除浏览器缓存
   - 按 Ctrl + Shift + Delete
   - 选择"缓存的图片和文件"
   - 点击"清除数据"

3. 换一个浏览器试试
   - 推荐：Chrome、Edge、Firefox
   - 不要用IE浏览器

4. 检查文件是否完整
   - 文件大小应该约277KB
   - 如果文件损坏，重新下载
```

---

### Q3：如何添加新国家？

**问题：** 我想在免签国家列表中添加一个新国家

**解决方案：**

```
1. 用代码编辑器打开 index.html
   （推荐：VS Code、Notepad++）

2. 按 Ctrl + F 搜索"📊 数据配置区"

3. 找到 visaFreeCountries 数组

4. 在对应大洲区域添加一行：
   
   // 例如添加越南（亚洲）
   { code: 'VN', zh: '越南', en: 'Vietnam', continent: 'asia' },

5. 保存文件（Ctrl + S）

6. 刷新浏览器（Ctrl + F5）

7. 测试：
   - 点击"免签入境"
   - 点击"亚洲"标签
   - 查看是否有"越南"
```

**格式说明：**
- `code`：国家代码（2个大写字母，如 VN、US）
- `zh`：中文名
- `en`：英文名
- `continent`：所属大洲（europe/asia/americas/oceania）

### Q4：如何修改政策文本？

**问题：** 我想修改免签政策的说明文字

**解决方案：**

```
1. 打开 index.html 文件

2. 搜索"🌍 国际化数据区"

3. 找到 translations 对象

4. 修改中文：
   
   zh: {
       vfEntryPolicyText: '新的中文政策文本...',
   }

5. 修改英文：
   
   en: {
       vfEntryPolicyText: 'New English policy text...',
   }

6. 保存文件，刷新浏览器

7. 测试：
   - 选择免签国家
   - 查看新文本
   - 切换语言测试
```

### Q5：功能出错了怎么办？

**问题：** 某个功能不工作了，或者报错了

**解决方案：**

```
第1步：查看错误信息
1. 按 F12 打开开发者工具

第3步：寻求帮助
1. 复制错误信息
2. 描述问题现象
3. 说明什么时候开始的
4. 使用"场景4：修复问题"模板发给AI
```

### Q8：怎么判断AI改得好不好？

**问题：** 我不知道AI的修改质量如何

**解决方案：**

检查以下5个点：

```
1. ✅ 功能正常
   - 所有功能都能用
   - 没有新的bug

2. ✅ 没有错误
   - 按F12看控制台
   - 没有红色错误

3. ✅ 代码整洁
   - 有清晰的注释
   - 代码格式规范

4. ✅ 白话解释
   - AI用日常语言解释了修改
   - 你能理解改了什么

5. ✅ 测试步骤
   - AI提供了详细的测试方法
   - 你能按照步骤验证
```

**如果有一点不满足，要求AI重新改。**

## 技术问题

### Q10：如何查看浏览器控制台？

**问题：** 我不知道怎么打开控制台看错误

**解决方案：**

```
Windows系统：
1. 在网页上按 F12
2. 或者右键点击页面 → "检查"
3. 点击"Console"（控制台）标签

Mac系统：
1. 按 Cmd + Option + I
2. 或者右键点击页面 → "检查"
3. 点击"Console"（控制台）标签

查看错误：
- 红色文字 = 错误
- 黄色文字 = 警告
- 白色/蓝色文字 = 普通信息
```

---

### Q11：如何备份文件？

**问题：** 我不知道怎么备份文件

**解决方案：**

```
方法1：复制粘贴
1. 右键点击 index.html
2. 选择"复制"
3. 在空白处右键选择"粘贴"
4. 重命名为：index-备份-日期.html
   例如：index-备份-20240415.html

方法2：另存为
1. 用代码编辑器打开文件
2. 选择"文件" → "另存为"
3. 输入新文件名
4. 保存

建议的命名规范：
- index-备份-20240415.html（按日期）
- index-添加越南-20240415.html（按修改内容）
```

### Q13：什么是"分区注释"？

**问题：** 文档里经常提到"分区注释"，是什么？

**解决方案：**

分区注释是代码中的标记，帮助你快速找到对应区域：

```
📄 index.html
├── 🎨 全局样式区 ← 这是分区注释
├── 📦 组件样式区 ← 这是分区注释
├── 🌐 页面主体区 ← 这是分区注释
│   ├── 🛠️ 核心工具区
│   ├── 🛂 免签政策区
│   └── 🏠 生活在中国区
└── 💻 JavaScript代码区 ← 这是分区注释
    ├── 📊 数据配置区 ← 改数据看这里
    └── 功能函数区
```

**好处：**
- 找东西快
- 修改安全
- 新人友好

## 数据更新问题

### Q14：更新数据后不显示怎么办？

**问题：** 我添加了新国家，但页面上看不到

**解决方案：**

```
检查清单：

1. 文件保存了吗？
   - 按 Ctrl + S 保存

2. 浏览器刷新了吗？
   - 按 Ctrl + F5 强制刷新

3. 格式正确吗？
   - 检查逗号、引号、括号
   - 对照现有格式

4. 代码写对位置了吗？
   - 确认在 visaFreeCountries 数组里
   - 不要写到函数里面

5. 有语法错误吗？
   - 按F12看控制台
   - 修复红色错误
```

### Q15：修改政策文本后切换语言不正常？

**问题：** 我改了中文，但切换英文后有问题

**解决方案：**

```
1. 检查中英文是否都改了
   
   translations = {
       zh: {
           vfEntryPolicyText: '中文文本', // 改了
       },
       en: {
           vfEntryPolicyText: 'English text', // 也要改
       }
   }

2. 检查引号是否匹配
   - 开始和结束都要有引号
   - 中文引号和英文引号不要混用

3. 检查特殊字符
   - 如果有引号，需要转义：\'
   - 换行用：\n

4. 保存并刷新
   - Ctrl + S 保存
   - Ctrl + F5 刷新
   - 测试两种语言
```

**问题：** 我想一次添加多个国家，可以吗？

**解决方案：**

可以！批量添加的方法：

## 部署问题

### Q17：如何部署到服务器？

**问题：** 我想把项目放到网上，让别人访问

**解决方案：**

```
方法1：使用GitHub Pages（免费）

1. 创建GitHub账号
2. 创建新仓库
3. 上传 index.html
4. 开启 GitHub Pages
5. 访问：https://用户名.github.io/仓库名

方法2：使用Netlify（免费）

1. 注册Netlify账号
2. 拖拽文件夹到Netlify
3. 自动部署完成
4. 获得访问链接

方法3：使用自己的服务器

1. 上传 index.html 到服务器
2. 配置Web服务器（Nginx/Apache）
3. 设置域名
4. 完成
```

---

### Q18：部署后访问很慢怎么办？

**问题：** 部署后页面加载很慢

**解决方案：**

```
优化方法：

1. 压缩图片
   - logo.png 可以用工具压缩
   - 推荐：TinyPNG

2. 启用Gzip压缩
   - 在服务器配置Gzip
   - 可以减少70%文件大小

3. 使用CDN
   - 使用Cloudflare等CDN服务
   - 加速全球访问

4. 启用浏览器缓存
   - 配置HTTP缓存头
   - 减少重复加载
```

### Q19：如何绑定自己的域名？

**问题：** 我想用自己的域名访问项目

**解决方案：**

```
以GitHub Pages为例：

1. 购买域名
   - 在阿里云、腾讯云等购买

2. 配置DNS
   - 添加CNAME记录
   - 指向：用户名.github.io

3. 配置GitHub Pages
   - 在仓库Settings中
   - 添加Custom domain
   - 输入你的域名

4. 等待生效
   - DNS生效需要几分钟到几小时
   - 测试访问
```
