## 文件分区架构

### 整体结构

项目采用**单文件架构**，所有代码都在 `index.html` 中，但通过清晰的分区注释组织：

```
📄 index.html（约 4374 行）
│
├── 🎨 全局样式区（CSS，位于 <style> 块）
│   ├── CSS 变量、基础样式、响应式设计
│   └── “小工具箱抽屉”等组件样式
│
├── 🌐 页面主体区（HTML <body>）
│   ├── 语言切换按钮、页面头部
│   ├── 主内容区 <main>（语义化聚合所有功能模块）
│   │   ├── 🛂 免签政策面板（免签入境 / 240h / 24h / 区域性）
│   │   ├── 🏠 生活服务区、法律合规提醒
│   │   └── 其他模块
│   ├── 🛠️ 小工具箱抽屉（`#tools-drawer`）
│   │   ├── 签证计算器、时区转换器、汇率换算器
│   │   └── 便捷问答（`#tool-pane-phrase`）
│   └── 页脚
│
└── 💻 JavaScript 代码区（位于 <script> 块内，均有明确注释分区）
    ├── 📌 初始化区 (INITIALIZATION) ～行 1773
    │   └── dayjs 加载探测、GA 上报等
    │
    ├── 🌍 国际化数据区 (INTERNATIONALIZATION) ～行 1790
    │   └── `translations` 对象（zh / en 两套文案）
    │
    ├── 🔧 全局变量区 (GLOBAL VARIABLES) ～行 2146
    │   ├── `vfSelectedCountry` / `vf240SelectedCountry`（当前选中项）
    │   └── 汇率缓存、定时器等
    │
    ├── 📊 数据配置区 (DATA CONFIGURATION ZONE) ～行 2182 ⭐ 重点
    │   ├── `tzs`（时区）
    │   ├── `visaFreeCountries`（免签国家）
    │   ├── `transit240Countries`（240h 适用国家）
    │   └── `transit240Provinces`（240h 开放省份/口岸）
    │
    ├── 🚀 页面初始化区 (PAGE INITIALIZATION) ～行 2541
    │   └── DOMContentLoaded、默认值填充、事件绑定
    │
    ├── 🛠️ 核心工具函数区 (CORE TOOL FUNCTIONS) ～行 2855
    │   ├── `calculateVisa()`、`convertTime()`、`convertCurrency()`
    │   ├── `renderTimezoneOptions()`
    │   ├── 免签模块：`switchVfTab()`、`searchCountry()`、`selectVfCountry()`
    │   └── 240h 过境模块：`renderTransit240Data()`、`search240Country()`、`select240CountryByCode()`
    │
    └── 💬 便捷问答模块 (QUICK PHRASEBOOK) ～行 3727
        ├── `QP_SCENES`（10 个场景定义）
        ├── `QP_BUILTIN`（每场景 3 条内置句子）
        └── `qpRenderScenes()` / `qpRenderPanelList()` / `qpRenderFs()`
```

> ⚠️ **与旧文档的区别**：之前文档提到的 `transit240Ports` / `transit24Ports` / `regionalVisaFree` / `renderVfCountryGrid` 均为**误写**。实际 `index.html` 中：
> - 240h 过境适用国家叫 `transit240Countries`，口岸/省份叫 `transit240Provinces`。
> - **24h 过境免签** 和 **区域性免签** 没有独立的 JS 数据数组，内容直接写在 HTML 面板 `#vf-panel-regional` 等中，多语言文案通过 `data-i18n` 从 `translations` 调取。
> - 免签国家渲染主函数为 `selectVfCountry()` / `searchCountry()`，不存在 `renderVfCountryGrid()`。
    │   │   └── 复制功能
    │   │
    │   ├── 生活服务模块函数区
    │   │   └── 生活服务相关函数
    │   │
    │   └── 法律合规模块函数区
    │       └── 法律合规相关函数
    │
    └── 工具函数区
        ├── 通用工具函数

```

## 关键数据配置

### 1. 免签国家列表

**变量名：** `visaFreeCountries`

**位置：** `📊 数据配置区`

**数据结构：**
```javascript
const visaFreeCountries = [
    { code: 'FR', zh: '法国', en: 'France', continent: 'europe' },
    { code: 'DE', zh: '德国', en: 'Germany', continent: 'europe' },
    { code: 'JP', zh: '日本', en: 'Japan', continent: 'asia' },
    // ... 共50个国家
];
```

**字段说明：**
- `code` - 国家代码（2个大写字母，ISO标准）
- `zh` - 中文名称
- `en` - 英文名称
- `continent` - 所属大洲（europe/asia/americas/oceania）

**如何添加新国家：**
```javascript
// 在对应大洲区域添加一行
{ code: 'VN', zh: '越南', en: 'Vietnam', continent: 'asia' },
```

### 2. 240h 过境免签适用国家与开放省份

**变量名：** `transit240Countries`（40 多个适用国家）、`transit240Provinces`（开放省份/口岸）

**位置：** `📊 数据配置区`（紧接在 `visaFreeCountries` 之后）

**数据结构：**
```javascript
const transit240Countries = [
    { code: 'US', zh: '美国', en: 'United States', continent: '...' },
    // ... 适用 240h 过境的国家
];

const transit240Provinces = [
    { id: 'beijing', zh: '北京市', en: 'Beijing', ... },
    // ... 开放 240h 过境的省份与口岸
];
```

> ⚠️ 不要写成 `transit240Ports`——该名在代码中**并不存在**。

### 4. 时区数据

**变量名：** `tzs`

const tzs = [

## 核心函数说明

### 1. calculateVisa() - 签证计算

**功能：** 计算最晚离境日期

**参数：**
- `entryDate` - 入境日期
- `stayDays` - 可停留天数

**返回值：** 最晚离境日期

**使用示例：**
```javascript
const entryDate = new Date('2024-01-01');
const stayDays = 30;
const departureDate = calculateVisa(entryDate, stayDays);
// 返回：2024-01-30
```

**位置：** 免签模块函数区

---

### 2. convertTime() - 时区转换

**功能：** 在不同时区之间转换时间

**参数：**
- `sourceTime` - 源时间
- `sourceTimezone` - 源时区
- `targetTimezone` - 目标时区

**返回值：** 转换后的时间

**使用示例：**
```javascript
const beijingTime = new Date('2024-01-01 15:00');
const nyTime = convertTime(beijingTime, 'Asia/Shanghai', 'America/New_York');
// 返回：2024-01-01 02:00 (纽约时间)
```

**位置：** 核心工具函数区

---

### 3. convertCurrency() - 汇率换算

**功能：** 货币兑换计算

**参数：**
- `amount` - 金额
- `fromCurrency` - 源货币
- `toCurrency` - 目标货币

**返回值：** 兑换后的金额

**使用示例：**
```javascript
const cnyAmount = convertCurrency(1000, 'USD', 'CNY');
// 返回：7200 (假设汇率为7.2)
```

**位置：** 核心工具函数区

**特点：**
- 调用外部API获取实时汇率
- 包含汇率缓存机制
- 支持多种货币

---

### 4. selectVfCountry() / searchCountry() - 免签国家交互

**功能：** 处理免签入境国家的选中与搜索

**位置：** 免签模块函数区（核心工具函数区内）

**作用：**
- 读取 `visaFreeCountries` 数据
- 在下拉/面板中过滤、展示国家列表
- 选中后填充详情与计算适用范围

**相关函数：**
- `selectVfCountry(code)` / `selectVfCountryByCode(code)`——选中某个免签国家
- `searchCountry(query)` / `searchCountryInDropdown(query)`——搜索过滤
- `switchVfTab(tabName)`——在免签入境/240h/24h/区域性面板之间切换

> ⚠️ **不存在** `renderVfCountryGrid()`——老文档误写，实际代码中由以上函数联合实现。

---

### 5. switchLanguage() - 切换语言

**功能：** 中英文切换

**参数：**
- `lang` - 目标语言（'zh' 或 'en'）

**作用：**
- 读取 `translations` 对象
- 替换页面所有文本
- 保持当前操作状态

**位置：** 界面交互函数区

## 💡 代码优化亮点

### 1. 数据与逻辑分离

**之前的问题：**
- 国家列表散落在多处
- 更新数据需要改好几个地方
- 容易改错

**现在的方案：**
```
📊 数据配置区（就在这里改！）
├── 免签国家列表（50个国家）
├── 240h 适用国家列表（transit240Countries）
├── 240h 开放省份/口岸列表（transit240Provinces）
└── 政策文本（中英文对照）

💻 代码逻辑区（不要动！）
├── 怎么显示国家列表
├── 怎么判断是否符合免签
└── 怎么切换中英文
```

---

### 2. 变量声明顺序优化

**之前的问题：**
- 变量在使用后才声明
- 导致"Temporal Dead Zone"错误
- 功能时灵时不灵

**现在的方案：**
```
🔧 全局变量区（所有工具都在这里！）
├── 汇率缓存工具
├── 定时器工具
├── 当前筛选状态工具
└── 当前选中国家工具

💻 工作流程区（按顺序使用工具）
1. 先从"全局变量区"拿到工具
2. 再用工具干活
3. 不会出错
```

---

### 3. 全中文注释

**之前：**
```javascript
// Render country grid
function selectVfCountry(code) { ... }
```

**现在：**
```javascript
// 选中免签国家并填充详情
function selectVfCountry(code) { ... }
```

**好处：**
- 一看就懂，不需要翻译
- 减少误解
- 协作顺畅
