### ⚠️ 修改铁律

1. **改前必读** [代码结构](./代码结构.md) 中的「文件分区架构」，明确改动落在哪个分区。
2. **数据与逻辑分离**：增删国家/口岸只改数据数组，**不要**改函数逻辑。
3. **多语言同步**：任何 UI 文案改动必须同时改 `translations.zh` 和 `translations.en`。
4. **测试三件套**：每次改动后必须验证「语言切换、复制按钮、签证/时区/汇率」三项核心功能未受影响。
5. **回滚预案**：修改前提示用户备份 `index.html`，告知具体回滚步骤。

## 🎯 项目简介

**waiguoren1** 是一个专为在中国生活或计划来华的外国人设计的综合性服务网页应用。项目提供签证计算、时区转换、汇率换算、免签政策查询等实用工具，帮助外国用户快速了解和使用中国的生活服务。

### 核心特点

- 🛠️ **实用工具**：签证计算器、时区转换器、汇率换算器
- 🛂 **政策查询**：免签入境、过境免签、区域性免签
- 🏠 **生活指南**：支付、外卖、出行、法律合规
- 🌍 **双语支持**：完整的中英文切换功能
- 📱 **单页应用**：纯 HTML/CSS/JavaScript，无需后端

## 🎯 核心要点

### 记住这5句话

1. **说效果，不说代码** - "让用户选择后自动更新" vs "添加onchange事件"
2. **说清楚不能影响什么** - "不能影响签证、汇率、时区功能"
3. **要求白话解释** - "请用日常语言解释，不要用技术术语"
4. **要求测试步骤** - "教我一步步测试，告诉我每一步看什么"
5. **要求撤销方法** - "如果出问题，怎么恢复到修改前"

### 记住这个流程

```
描述需求 → AI修改 → 提供测试步骤 → 你测试验收 → 通过/重做
```

### 记住这个心态

```
你是产品经理，AI是程序员。
你描述需求，AI负责实现。
你验收质量，AI负责修复。
```
