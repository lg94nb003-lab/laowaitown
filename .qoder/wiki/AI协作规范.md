## 标准化提问模板

> 💡 **使用说明**：点击代码框右上角的"复制"按钮，然后粘贴到AI对话框，填入你的需求即可。

---

### 完整模板（推荐）

```
# 【需求标题】[用一句话描述你要做什么]

## 【现状描述】
[描述现在的情况，越具体越好]

包括：
- 功能在哪里（页面哪个位置）
- 现在是怎么工作的
- 如果有相关代码位置，也说明（如"数据配置区"）

## 【希望效果】
[描述你想要的最终效果，用日常语言，不要用技术术语]

分点说明：
1. 第一点效果
2. 第二点效果
3. 第三点效果

## 【特别重要的要求】
⚠️ **改完后必须检查：**
- [ ] [功能A]还能正常用（例如：签证计算器）
- [ ] [功能B]还能正常用（例如：汇率换算）
- [ ] [功能C]还能正常用（例如：中英文切换）
- [ ] 按F12看控制台，没有红色错误
- [ ] 刷新页面，所有功能都正常显示
- [ ] [其他你关心的功能]

## 【你怎么交付给我】
1. 用白话告诉我改了哪里（不要用技术术语）
2. 教我一步步测试（例如：先刷新页面，然后点XX，看XX）
3. 告诉我如果出问题怎么恢复
4. 如果涉及多个方案，请先给我选择

## 【我不懂的术语请用白话解释】
如果有技术概念，请用白话解释。

要求：
- 不要说"添加onchange事件"
- 要说"让下拉框选择后自动更新结果"
- 不要说"修改DOM结构"
- 要说"改变页面的显示方式"
```

## 5个黄金法则

### 法则1：说"我要什么效果"，不说"我要改什么代码"

❌ **错误说法：**
```
"帮我给select标签加一个onchange事件，调用convertCurrency函数"
```

✅ **正确说法：**
```
"让用户切换货币时，自动更新兑换结果，不用点按钮"
```

**为什么：**
- 你是产品经理，不是程序员
- 描述效果，让AI决定技术方案
- 避免限制AI的解决思路

---

### 法则2：说清楚"不能影响什么"

❌ **错误说法：**
```
"帮我修改免签功能"
```

✅ **正确说法：**
```
"帮我修改免签功能的国家选择方式，但是：
- 不能影响签证计算器
- 不能影响汇率换算
- 不能影响中英文切换"
```

**为什么：**
- AI不知道哪些功能重要
- 明确边界，避免误伤
- 减少返工次数

---

### 法则3：要求AI用白话解释

❌ **错误做法：**
```
AI说："添加了addEventListener监听input事件"
你看不懂，但不敢问
```

✅ **正确做法：**
```
直接说："请用白话解释你改了什么，不要用技术术语"
```

**示例：**
- ❌ AI说："添加了addEventListener"
- ✅ AI应该说："让输入框在用户打字时自动更新结果"

**为什么：**
- 你需要理解改了什么
- 方便以后自己修改
- 验证AI是否真的理解了需求

---

### 法则4：要求分步骤测试

❌ **错误做法：**
```
AI说："改完了，功能正常"
你不确定，但不知道该怎么测试
```

✅ **正确做法：**
```
要求AI："教我一步步测试，告诉我每一步应该看到什么效果"
```

**示例：**
```
✅ 好的测试步骤：
1. 刷新页面 → 看到"免签政策"标题
2. 点"全部"标签 → 看到50个国家方块
3. 点"德国" → 看到绿色政策框
4. 切换英文 → 政策框变成英文

❌ 不好的测试步骤：
"测试一下功能是否正常"
```

**为什么：**
- 确保你真的能验证
- 避免遗漏重要测试
- 发现问题更快

---

### 法则5：要求提供"撤销方法"

❌ **错误做法：**
```
改完后发现有问题，但不知道怎么办
```

✅ **正确做法：**
```
要求AI："如果出问题，告诉我怎么恢复到修改前的状态"
```

**为什么：**
- 给自己留后路
- 减少修改风险
- 增强信心

## 协作流程

### 标准流程

```
第1步：描述需求
  ↓
第2步：AI修改代码
  ↓
第3步：AI提供测试步骤
  ↓
第4步：你测试验收
  ↓
第5步：通过 / 重做
```

---

### 详细步骤

#### 第1步：描述需求

```
1. 复制合适的模板（完整/简化/场景）
2. 填入你的需求
3. 检查是否清晰完整
4. 发给AI
```

**检查清单：**
- [ ] 需求标题清晰
- [ ] 现状描述具体
- [ ] 希望效果明确
- [ ] 列出了不能影响的功能
- [ ] 要求了测试步骤和白话解释

---

#### 第2步：AI修改代码

```
1. 等待AI完成修改
2. AI会告诉你改了哪里
3. AI会提供测试步骤
```

**注意：**
- 不要催促AI
- 如果有疑问，立即提出
- 要求AI先说明影响范围

---

#### 第3步：测试验收

```
1. 备份当前文件
2. 用AI修改后的文件替换
3. 刷新浏览器（Ctrl + F5）
4. 按AI提供的步骤测试
5. 按F12检查控制台
```

**测试清单：**
- [ ] 刷新页面正常
- [ ] 控制台无红色错误
- [ ] 新功能正常
- [ ] 旧功能正常
- [ ] 中英文切换正常

---

#### 第4步：验收结果

**通过：** ✅
```
- 所有测试通过
- 保存修改
- 记录修改日志
- 继续下一个需求
```

**不通过：** ❌
```
- 截图错误信息
- 描述具体问题
- 要求AI修复
- 重新测试
```

---

### 分阶段开发流程

对于复杂功能，建议分阶段：

```
第一阶段：基本功能
"先做一个能用的版本，不用太好看"
  ↓ 测试通过
第二阶段：优化效果
"现在优化一下样式，让它更好看"
  ↓ 测试通过
第三阶段：处理细节
"最后处理边界情况和错误提示"
  ↓ 测试通过
完成！
```

**好处：**
- 每阶段都能测试
- 出问题容易定位
- 不会一次改太多

## 常见问题解答

### 代码结构相关

#### Q1：我可以自己改代码吗？

**答：** 

- ✅ **可以改的地方：** "📊 数据配置区"（国家列表、政策文本）
- ❌ **不要改的地方：** "💻 代码逻辑区"（函数、事件处理）

**为什么：**
- 数据配置区：照着格式改就行，像填表格
- 代码逻辑区：需要理解编程逻辑，改错会导致功能出错

**安全做法：**
- 改之前，先备份文件
- 改之后，刷新页面测试
- 有问题就恢复备份

---

#### Q2：如果我不小心改坏了怎么办？

**答：** 三步恢复法：

```
1. 立即停止编辑 - 不要继续改，避免更乱
2. 恢复备份 - 用之前备份的文件覆盖
3. 重新告诉AI需求 - 用模板描述需求，让AI重新改
```

**预防方法：**
- 每次修改前都备份
- 小步快跑：一次改一个小功能
- 测试通过再改下一个

---

#### Q3：我怎么知道代码有没有错误？

**答：** 三步检查法：

```
1. 按F12打开开发者工具
2. 点击"Console"（控制台）标签
3. 看有没有红色文字
```

- ✅ **没有红色** = 正常
- ❌ **有红色** = 有错误，把错误信息截图发给AI

**示例：**
```
✅ 正常情况：
🚀 Initializing visa-free module...
🔍 selectVfCountry called

❌ 错误情况：
Uncaught ReferenceError: xxx is not defined
```

### AI协作相关

#### Q4：AI改的代码我看不懂怎么办？

**答：** 直接要求AI用白话解释：

**你可以说：**
- "请用小学生都能听懂的话解释你改了什么"
- "不要用技术术语，用日常语言说"
- "打个比喻告诉我这个功能是干嘛的"

**示例：**
- ❌ AI说："添加了addEventListener监听input事件"
- ✅ AI应该说："让输入框在用户打字时自动更新结果"

---

#### Q5：AI改完后功能坏了怎么办？

**答：** 四步排查法：

```
1. 按F12看控制台 - 有没有红色错误
2. 截图错误信息 - 发给AI
3. 描述现象 - "点击XX没反应"、"显示XX错误"
4. 要求AI修复 - "请修复这个问题，并告诉我怎么测试"
```

**预防方法：**
- 改之前要求AI列出"影响范围"
- 改之后立即测试所有相关功能
- 每次只改一个小功能

---

#### Q6：怎么判断AI改得好不好？

**答：** 五个检查点：

```
1. ✅ 功能正常 - 所有功能都能用
2. ✅ 没有错误 - 控制台没有红色错误
3. ✅ 代码整洁 - 有清晰的注释和分区
4. ✅ 白话解释 - AI用日常语言解释了修改
5. ✅ 测试步骤 - AI提供了详细的测试方法
```

**如果有一点不满足，要求AI重新改。**

### 数据更新相关

#### Q7：怎么添加新国家？

**答：** 三步添加法：

```
1. 打开文件 - 用记事本或代码编辑器打开index.html
2. 找到位置 - 搜索"📊 数据配置区"，找到visaFreeCountries
3. 添加一行 - 照着现有格式加一行
```

**示例：**
```javascript
// 在亚洲国家区域添加：
{ code: 'VN', zh: '越南', en: 'Vietnam', continent: 'asia' },
{ code: 'LA', zh: '老挝', en: 'Laos', continent: 'asia' },
```

**格式说明：**
- `code`：国家代码（2个大写字母）
- `zh`：中文名
- `en`：英文名
- `continent`：所属大洲（europe/asia/americas/oceania）

**添加后：**
- 保存文件
- 刷新浏览器
- 测试：点"亚洲"标签，看有没有新国家

---

#### Q8：怎么修改政策文本？

**答：** 三步修改法：

```
1. 打开文件 - 打开index.html
2. 找到位置 - 搜索"🌍 国际化数据区"，找到translations
3. 修改文本 - 改zh（中文）或en（英文）对应的文本
```

**示例：**
```javascript
translations = {
    zh: {
        vfEntryPolicyText: '新的中文政策文本...',
    },
    en: {
        vfEntryPolicyText: 'New English policy text...',
    }
}
```

**修改后：**
- 保存文件
- 刷新浏览器
- 测试：选德国 → 看新文本 → 切换英文 → 看英文文本
