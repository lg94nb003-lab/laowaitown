## 🎯 第一次使用（5步上手）

### 第1步：打开文档

找到这两个文件：
- 📘 `AI协作编程指南.md` - 完整手册（需要时查阅）
- 🚀 `快速使用指南.md` - 本文档（快速上手）

### 第2步：复制提问模板

打开《AI协作编程指南》，找到"第三部分"，复制这个模板：

```markdown
# 【需求标题】[用一句话描述]

## 【现状描述】
[描述现在的情况]

## 【希望效果】
[描述你想要的效果，用日常语言]

## 【特别重要的要求】
⚠️ **改完后必须检查：**
- [ ] [功能A]还能正常用
- [ ] [功能B]还能正常用
- [ ] 按F12没有红色错误

## 【你怎么交付给我】
1. 用白话告诉我改了哪里
2. 教我一步步测试
3. 告诉我如果出问题怎么恢复
```

### 第3步：填入你的需求

**示例：添加新国家**

```markdown
# 【需求标题】添加越南到免签国家列表

## 【现状描述】
现在免签国家有50个，在代码的"数据配置区"。

## 【希望效果】
在亚洲国家里添加越南，用户选越南后显示绿色免签政策框。

## 【特别重要的要求】
⚠️ **改完后必须检查：**
- [ ] 免签功能正常
- [ ] 搜索功能正常
- [ ] 中英文切换正常
- [ ] 按F12没有红色错误

## 【你怎么交付给我】
1. 用白话告诉我改了哪里
2. 教我测试：刷新页面 → 点"亚洲" → 看有没有越南
3. 告诉我如果出问题怎么恢复
```

### 第4步：发给AI

把填好的模板发给AI，等待修改完成。

### 第5步：测试验收

按照AI提供的测试步骤，一步步测试：

1. ✅ 刷新页面
2. ✅ 检查功能
3. ✅ 按F12看控制台
4. ✅ 确认没有问题

**全部通过 = 验收成功！** 🎉

## 💡 常用场景速查

### 场景1：添加/修改数据

**适用：** 添加国家、修改政策文本、更新联系方式

**做法：**
```
1. 告诉AI要添加什么数据
2. 说明在哪个区域修改（数据配置区）
3. 要求AI提供格式示例
4. 测试：刷新页面 → 查看新数据
```

**示例提问：**
```
【需求】添加越南、老挝到免签国家列表
【现状】现在有50个国家，在数据配置区的visaFreeCountries数组
【希望】在亚洲区域添加越南(VN)、老挝(LA)
【要求】提供添加格式，教我测试
```

---

### 场景2：修改功能效果

**适用：** 改变按钮颜色、调整布局、修改提示文本

**做法：**

2. 说明在哪个模块

4. 要求AI提供前后对比
```

**示例提问：**

【现状】现在选法国显示绿色背景的政策框

【不能影响】不能影响国家选择、搜索、中英文切换
【要求】改完后教我测试，提供截图对比
```

---

### 场景4：修复问题

**适用：** 功能不工作、显示错误、控制台报错

**做法：**
```
1. 描述现象（哪里不对）
2. 提供错误截图（如果有）
3. 说明什么时候开始出错的
4. 要求AI修复并解释原因
```

**示例提问：**

【要求】修复问题，用白话解释原因，教我测试

## ⚠️ 常见错误及解决方法

### 错误1：改完后功能坏了

**原因：** AI修改时影响了其他功能

**解决：**
1. 立即停止编辑
2. 恢复备份文件
3. 重新描述需求，强调"不能影响XX功能"
4. 要求AI先说明影响范围，再修改

**预防：**
- 每次修改前都备份
- 要求AI列出影响范围
- 一次只改一个小功能

---

### 错误2：看不懂AI的解释

**原因：** AI用了技术术语

**解决：**
直接说：
```
"请用小学生都能听懂的话重新解释"
"不要用技术术语，打个比喻告诉我"
"用日常语言说，不要说代码"
```

**示例：**
- ❌ AI说："添加了addEventListener"
- ✅ AI应该说："让按钮在点击后自动执行某个功能"

---

### 错误3：测试不知道看什么

**原因：** AI没提供详细的测试步骤

**解决：**
要求AI：
```
"请提供详细的测试步骤，告诉我：
1. 第一步做什么，应该看到什么
2. 第二步做什么，应该看到什么
3. 怎样算成功，怎样算失败"
```

**示例：**
```
✅ 好的测试步骤：
1. 刷新页面 → 看到"免签政策"标题
2. 点"全部"标签 → 看到50个国家方块
3. 点"德国" → 看到绿色政策框
4. 切换英文 → 政策框变成英文

❌ 不好的测试步骤：
"测试一下功能是否正常"
```

---

### 错误4：改了好几次还是不对

**原因：** 需求描述不够清晰

**解决：**
1. 停下来，重新整理需求
2. 用模板重新描述
3. 提供截图或示例
4. 分步骤实现，不要一步到位

**技巧：**
```
第一步：先实现基本功能
第二步：再优化效果
第三步：最后处理细节

不要一次性要求太多，容易出错。
```

## 🔥 高级技巧

### 技巧1：分阶段开发

不要一次要求太多，分阶段进行：

```
第一阶段：实现基本功能
"先做一个能用的版本，不用太好看"

第二阶段：优化效果
"现在优化一下样式，让它更好看"

第三阶段：处理细节
"最后处理边界情况和错误提示"
```

**好处：**
- 每阶段都能测试
- 出问题容易定位
- 不会一次改太多

---

### 技巧2：要求AI提供选择

让AI给你方案，你来做决定：

```
"我有这个需求，请给我2-3个实现方案：
方案1：简单快速，但功能有限
方案2：功能完整，但复杂一些
方案3：折中方案

请说明每个方案的优缺点，我来选择。"
```

**好处：**
- 你掌控决策权
- 了解不同选择
- 避免AI自作主张

---

### 技巧3：建立修改日志

记录每次修改，方便回溯：

```
修改日志：
2024-04-15：添加越南、老挝到免签国家列表
2024-04-16：修改政策文本
2024-04-17：新增紧急联系方式模块

每次修改前备份文件，命名为：
index-备份-20240415.html
```

**好处：**
- 知道改了什么
- 出问题能快速恢复
- 积累经验

---

### 技巧4：学会说"不"

如果AI改得不好，直接要求重做：

```
"这个方案不好，请重新做：
1. 不要影响XX功能
2. 用YY方式实现
3. 先给我看方案，确认后再改"
```

**记住：**
- 你是产品经理，AI是程序员
- 你有权要求重做
- 不要勉强接受不好的方案

## 🎯 核心要点总结

### 记住这5句话

1. **说效果，不说代码** - "让用户选择后自动更新" vs "添加onchange事件"
2. **说清楚不能影响什么** - "不能影响签证、汇率、时区功能"
3. **要求白话解释** - "请用日常语言解释，不要用技术术语"
4. **要求测试步骤** - "教我一步步测试，告诉我每一步看什么"
5. **要求撤销方法** - "如果出问题，怎么恢复到修改前"

### 记住这个流程

```
描述需求 → AI修改 → 提供测试步骤 → 你测试验收 → 通过/重做
```

### 记住这个心态

```
你是产品经理，AI是程序员。
你描述需求，AI负责实现。
你验收质量，AI负责修复。
```
